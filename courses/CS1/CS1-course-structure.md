## Module overview

| # | Module | What students build / focus | Book project thread |
|---|--------|----------------------------|------------------------|
| 1 | GitHub Foundations | Personal repo + branch + PR | Same workflow for all later code |
| 2 | HCI and Figma | Needfinding + Figma (**Track A** Book, or **Track B** redesign existing UI) | Track A → Book UX; Track B → skills then return to Book |
| 3 | LaTeX | One-page technical doc + design rationale / summary | Documents scope and design decisions |
| 4 | HTML / CSS / Bootstrap | **Static responsive login page** | First static Book entry screen |
| 5 | Grid and Flexbox | **Same login page**, refined with **12-col grid + Flexbox** | Layout depth on login only |
| 6 | JavaScript (DOM) | **Dice Challenge** (vanilla JS, bounded UI) | DOM skills before Next; Book UI interactivity in Next |
| 7 | Next.js | Multi-page app, components, `fetch`, forms → **Book frontend** | Main Book UI |
| 8 | FastAPI | **Book CRUD API** | Backend contract |
| 9 | PostgreSQL | Persist Book data in **Postgres** | Replace in-memory / temp store |
| 10 | Final outcome | **Present full-stack Book app** + explanation | End-to-end demo |

---

## End product (Module 10)

A simple **full-stack Book application**: HCI input, Figma, Bootstrap **login page** (grid/flex), JS patterns applied in the app via Next, **Next.js** frontend, **FastAPI** backend, **PostgreSQL**, **GitHub** collaboration.
